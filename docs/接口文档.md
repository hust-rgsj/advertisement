# 互联网广告系统

## 1 企业管理员

### 1.1 管理用户账号权限

### 1.1.1 基本信息

> 请求路径：admin/customer/status
>
> 请求方式：POST
>
> 接口描述：该接口用于管理用户账号权限

### 1.1.2 请求参数

参数格式: query参数

| 参数名称       | 类型      | 是否必须 | 备注                |
|------------|---------|------|-------------------|
| status     | Integer | 必须   | 用户账号权限, 0-禁用，1-正常 | 
| customerId | Integer | 必须   | 用户id              |

请求数据样例：

```shell
  http://192.168.174.1:9526/customer/status?customerId=1933635587&status=0
  http://192.168.174.1:9526/customer/status?customerId=1933635587&status=1
```

### 1.1.3 响应数据

参数格式：application/json

参数说明：

| 参数名称 | 类型     | 是否必须 | 备注  |
|------|--------|------|-----|
| code | number | 必须   |     | 响应码, 200 成功 ; 500 失败 |      |
| msg  | string | 非必须  |     | 提示信息                 |      |
| data | object | 非必须  |     | 返回的数据                |      |

响应数据样例：

```json
{
  "code": 200,
  "msg": null,
  "data": "已解封该账号"
}
```

```json
{
  "code": 200,
  "msg": null,
  "data": "已成功禁用该账号"
}
```

### 1.2 企业广告审核

#### 1.2.1 基本信息

> 请求路径：/admin/advertisement/examine
>
> 请求方式：POST
>
> 接口描述：该接口用于企业管理员广告审核

#### 1.2.2 请求参数
参数格式 ：application/json

参数说明

| 参数名    | 类型     | 是否必须 | 备注                           |
|--------|--------|------|------------------------------|
| id     | number | 必须   | 广告id                         |
| status | number | 必须   | 审核状态：10-未审核，11-审核不通过，12-审核通过 |
| reason | string | 非必须  | 审核不通过原因                      |

请求参数样例：

```json
 {
  "advertisement_id": 1,
  "audit_status": 11,
  "reason": "广告审核不通过"
}
```

#### 1.2.3 响应数据

参数格式：application/json

响应数据样例：

```json
{
  "code": 200,
  "msg": null,
  "data": "审核通过"
}
```

### 2.1 查询广告展示情况

#### 2.1.1 基本信息

> 请求路径：/customer/advertisement/page
>
> 请求方式：GET
>
> 接口描述：该接口用于查询用户所持有的广告情况
>

参数格式：queryString

参数说明：

| 参数名称    | 是否必须 | 示例  | 备注                 |

| pageNum | 否    | 1   | 分页查询的页码，如果未指定，默认为1 |

```json
{
  "code": 200,
  "msg": "success",
  "data": "广告信息添加成功"
}
```

## 3 其他接口

### 3.1 登录

#### 3.1.1 基本信息

> 请求路径：/login
>
> 请求方式：POST
>
> 接口描述：该接口用于客户和管理员登录

#### 3.1.2 请求参数

参数格式：application/json

参数说明：

| 名称       | 类型     | 是否必须 | 备注  |
|----------|--------|------|-----|
| username | string | 必须   | 用户名 |
| password | string | 必须   | 密码  |

请求数据样例：

```json
{
  "username": "xiaoxiao",
  "password": "123456"
}
```

#### 3.1.3 响应数据

参数格式：application/json

参数说明：

| 名称            | 类型     | 是否必须 | 默认值 | 备注                   | 其他信息 |
|---------------|--------|------|-----|----------------------|------|
| code          | number | 必须   |     | 响应码, 200 成功 ; 500 失败 |      |
| msg           | string | 非必须  |     | 提示信息                 |      |
| data          | object | 必须   |     | 返回的数据                |      |
| Authorization | string | 必须   |     | token值               |
| id            | number | 必须   |     | 用户id                 |
| expire        | number | 必须   |     | 失效时间戳                |

响应数据样例：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9.eyJwYXNzd29yZCI6IjEyMzQ1NiIsImlkIjoxLCJ0eXBlIjpudWxsLCJ1c2VybmFtZSI6IlN3YWdneSIsImV4cCI6MTY4MzM2NDI1NX0.C65RqKXdb0ic2BCetOcJHdwureXvHUJ7O3fY4gWAUFc",
    "id": 1,
    "expire": 1683364255
  }
}
```

#### 3.1.4 备注说明

> 用户登录成功后，然后在后续的每次请求中，都需要在请求头header中携带token到服务端
>
> 如果检测到用户未登录，则会返回如下固定错误信息：
>
> ```json
> {
> 	"code": 500,
> 	"msg": "NOT_LOGIN",
>         "data": null
> }
> ```
