# 互联网广告系统

### 1 登录

#### 1.1 基本信息

> 请求路径：/login
>
> 请求方式：POST
>
> 接口描述：该接口用于客户和管理员登录

#### 1.2 请求参数

参数格式：application/json

参数说明：

| 名称     | 类型   | 是否必须 | 备注   |
| -------- | ------ | -------- | ------ |
| username | string | 必须     | 用户名 |
| password | string | 必须     | 密码   |

请求数据样例：

```json
{
  "username": "xiaoxiao",
  "password": "123456"
}
```

#### 1.3 响应数据

参数格式：application/json

参数说明：

| 名称     | 类型     | 是否必须 | 默认值 | 备注                   | 其他信息 |
|--------|--------|------|-----|----------------------|------|
| code   | number | 必须   |     | 响应码, 200 成功 ; 500 失败 |      |
| msg    | string | 非必须  |     | 提示信息                 |      |
| data   | object | 必须   |     | 返回的数据                |      |
| token  | string | 必须   |     | token值               |
| id     | number | 必须   |     |                      |
| type   | number | 必须   |     | 0：用户，1：管理员           |
| expire | number | 必须   |     | 时间戳(到期时间)            |

响应数据样例：

```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9.eyJwYXNzd29yZCI6IjAwMCIsImlkIjoxOTMzNjM1NTg3LCJ0eXBlIjowLCJ1c2VybmFtZSI6IjAwMCIsImV4cCI6MTY4MjUxMjg3M30.TAfWDTjqAoXwEN1xmuwvuyMjIGymf5uYCO5Kz997xxQ",
    "id": 1933635587,
    "type": 0,
    "expire": 1682512873
  }
}
```

#### 1.4 备注说明

> 用户登录成功后，系统会自动下发JWT令牌，然后在后续的每次请求中，都需要 在请求头header中携带到服务端，请求头的名称为 token ，值为登录时下发的JWT令牌。
>
> 如果检测到用户未登录，则会返回如下固定错误信息：
> ```json
> {
> 	"code": 500,
> 	"msg": "NOT_LOGIN",
> 	"data": null
> }
> ```
